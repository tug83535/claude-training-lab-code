### Technical dead-ends already hit
- **`SendKeys` against modal dialogs is fragile** — Gemini review confirmed 8 PASS / 50 FAIL. Use Path A silent wrappers (`DirectorXxx` subs) instead. Never propose SendKeys for dialog automation.
- **`Chr(9472)` and similar high-Unicode chars crash VBA** — VBA `Chr()` only handles 0–255. Use `String(50, "=")` or stick to ASCII. Lessons.md has this.
- **Pivot tables cannot be created via openpyxl** — must be created manually in Excel or via Copilot prompt.
- **`xlSheetVeryHidden` blocks hyperlinks** — use `xlSheetHidden` if hyperlinks need to navigate to the sheet.
- **`SpecialCells` `rng` not reset between loop iterations** — causes double-processing of previous sheet's cells. Always `Set rng = Nothing` between iterations.
- **`LogAction` 4th arg expects a String status, not a Double** — 13 instances of this bug have been found historically. If you write code that calls `LogAction`, the 4th arg is "OK" / "FAIL" / etc., NEVER an `elapsed` time.
- **Headers live on `HDR_ROW_REPORT` (row 4), not row 1** — multiple bugs from reading row 1 by mistake.
- **`StyleHeader` takes exactly 3 arguments** — not 4.

### Color / branding gotchas
- VBA `RGB()` uses BGR byte order historically; brand colors hardcoded inline in newer modules use correct values
- iPipeline Blue = `#0B4779` = `RGB(11, 71, 121)` — do NOT use `RGB(31, 73, 125)` (wrong)
- `CLR_HDR` was wrong (green channel 3 instead of 71) in 5 modules — already fixed in commit `63482a4`
- New visual/styling work uses official brand guide; existing modConfig color constants are NOT to be edited
