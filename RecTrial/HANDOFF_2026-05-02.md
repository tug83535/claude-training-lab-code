## 4. KEY DECISIONS AND RATIONALE (do not re-derive)

**All V4 direction locked 2026-04-28 — do not re-open:**
- Single video, 9–12 min — no 4a/4b split
- Hero: Revenue Leakage Finder narrative + ARR waterfall as closing visual
- Deliverable name: `finance_automation_launcher.py` (NOT `finance_copilot.py` — "Copilot" implies AI)
- Public surface: 5–7 workflows shown; full toolkit discoverable inside the workbook
- Adoption-grade: coworkers use on their own real files, not just watch

**VBA launcher — key architecture (confirmed working):**
- Uses `FSO.FileExists()` not `Dir()` — Dir() gives false positives on OneDrive Files On-Demand
- Uses `WScript.Shell` + `CurrentDirectory = ThisWorkbook.Path` + relative paths — avoids CMD quoting issues
- Full VBA code in `RecTrial\UniversalToolkit\python\ZeroInstall\modFinanceToolsLauncher.bas`

**Package structure — critical path resolution rule:**
- `samples\` must be inside `scripts\` (i.e. `scripts\samples\`) — NOT at the package root
- Reason: `safe_io.py` calculates `_TOOLKIT_ROOT = Path(__file__).resolve().parent.parent` = two levels up from `common\safe_io.py` = the `scripts\` folder. If samples is at the package root, the scripts can't find it.
- `outputs\` similarly resolves to `scripts\outputs\` at runtime (created automatically on first run)

**Opening the workbook:**
- Always open `FinanceTools.xlsm` from INSIDE `FinanceTools_v1.0\` — the VBA uses `ThisWorkbook.Path` for relative paths. Opening from elsewhere breaks Python path resolution.

## 5. GOTCHAS / DEAD ENDS ALREADY RULED OUT

- **Dir() for file existence checks on OneDrive** — gives false positives on Files On-Demand placeholders. Always use `FSO.FileExists()` in VBA for this project.
- **VBA Shell() with complex quoted paths** — fails with "filename, directory name, or volume label syntax is incorrect." Switch to `WScript.Shell` + `CurrentDirectory` + relative paths instead.
- **samples\ at the package root** — the scripts won't find it. Must be `scripts\samples\`. Spent debugging time on this; the fix is a folder location change, not a code change.
