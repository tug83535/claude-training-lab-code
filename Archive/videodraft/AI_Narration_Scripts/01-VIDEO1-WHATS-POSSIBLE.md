# Video 1 — "What's Possible" — AI Narration Script

**Runtime Target:** 4:00 to 5:00
**Audience:** All employees
**Purpose:** Build awareness, show 4 high-impact features, point to next steps
**Audio Segments:** 7 clips to generate in ElevenLabs

---

## How to Use This Script

1. Each **SEGMENT** below is one audio clip you generate in ElevenLabs
2. Copy only the text inside the **[PASTE INTO ELEVENLABS]** box
3. Generate the clip, download it, name it using the filename shown
4. The **[YOUR SCREEN ACTIONS]** section tells you what to do on screen while that audio plays
5. The **[TIMING NOTE]** tells you roughly how long each clip should be

## SEGMENT 1.1 — Opening Hook

**Filename:** `V1_S1_Opening_Hook.mp3`
**Timing:** ~25 seconds
**What's on screen:** Excel file open on the Report landing page

### [PASTE INTO ELEVENLABS]:

```
This is a single Excel file. Nothing to install, nothing to configure... you just open it and go.

Inside are sixty-two automated actions that handle reporting, analysis, data quality checks, charts, exports, and more... each one triggered with a single click.

In the next few minutes, I'm going to show you what that looks like.
```

### [YOUR SCREEN ACTIONS]:
1. Excel is already open on the Report page
2. As the voice starts, slowly scroll down the Report page — let the viewer see it's a real, populated workbook
3. Keep scrolling slowly through the whole clip — no clicking, just a smooth scroll
4. Stop scrolling right as the voice says "what that looks like"

### [TIMING NOTE]:
- Wait 2 seconds of silence before this clip starts (add in editor)
- The "sixty-two" should land clearly — if ElevenLabs rushes it, add a comma before it: "Inside are, sixty-two automated actions"

## SEGMENT 1.2 — Command Center Introduction

**Filename:** `V1_S2_Command_Center.mp3`
**Timing:** ~35 seconds
**What's on screen:** Opening and browsing the Command Center

### [PASTE INTO ELEVENLABS]:

```
Everything runs from one place... the Command Center.

You open it with Control Shift M, or from the button on the landing page.

Every action is organized by category... Monthly Operations, Analysis, Reporting, Enterprise Features, and more. You can scroll through to browse, or just search.

Type what you're looking for and it filters instantly. Find the action, click Run, and it handles the rest.

Let me show you a few examples.
```

### [YOUR SCREEN ACTIONS]:
1. **On "You open it with Control Shift M"** → press Ctrl+Shift+M to open the Command Center
2. Wait 2 seconds after it opens — let the viewer take it in
3. **On "organized by category"** → slowly scroll through the categories
4. **On "or just search"** → click the search box, type "variance" — show filtered results
5. Clear the search
6. **On "Let me show you a few examples"** → leave Command Center open

### [TIMING NOTE]:
- The pause after the Command Center opens is important. Don't rush to the next click.
- If ElevenLabs runs "Monthly Operations, Analysis, Reporting, Enterprise Features" too fast, add ellipses between them: "Monthly Operations... Analysis... Reporting... Enterprise Features"

## SEGMENT 1.3 — Data Quality Scan + Letter Grade

**Filename:** `V1_S3_Data_Quality.mp3`
**Timing:** ~40 seconds
**What's on screen:** Running Data Quality Scan from the Command Center

### [PASTE INTO ELEVENLABS]:

```
First... data quality. Before you do anything with your numbers, you want to know the data is clean.

One click, and it scans your entire workbook across six categories... completeness, accuracy, consistency, formatting, outliers, and cross-references.

It gives you a letter grade... right there at the top. You get a full breakdown underneath showing exactly where issues are, if any.

Fifteen seconds, start to finish.
```

### [YOUR SCREEN ACTIONS]:
1. **On "First, data quality"** → find Data Quality Scan in the Command Center
2. **On "One click"** → click Run on the Data Quality Scan action
3. Wait for the macro to complete — the Data Quality Report sheet appears
4. **On "letter grade, right there at the top"** → hover your cursor near the letter grade badge. Hold for 3 seconds.
5. **On "full breakdown underneath"** → slowly scroll down through the category breakdown
6. **On "Fifteen seconds"** → stop scrolling, pause

### [TIMING NOTE]:
- The letter grade badge is the visual anchor. Make sure you're hovering near it when the voice says "letter grade."
- After "Fifteen seconds, start to finish" — leave 2 seconds of silence. Let the speed sink in.
- If the macro takes longer than expected, that's OK. You'll adjust timing in the editor.

## SEGMENT 1.4 — Variance Commentary

**Filename:** `V1_S4_Variance_Commentary.mp3`
**Timing:** ~40 seconds
**What's on screen:** Running Variance Commentary from the Command Center

### [PASTE INTO ELEVENLABS]:

```
Next... one of the most useful features in the whole file.

After running a variance analysis, the system can automatically generate written commentary for the biggest variances.

These are plain English narratives... ready to drop into an email, a report, or a presentation. It identifies the line item, the dollar and percentage change, and describes what happened.

No copying numbers into a paragraph. No writing it yourself. One click.
```

### [YOUR SCREEN ACTIONS]:
1. **On "Next"** → open Command Center (Ctrl+Shift+M)
2. Find Variance Commentary action
3. **On "the system can automatically generate"** → click Run
4. Wait for the macro to complete — Variance Commentary sheet appears
5. **IMPORTANT: Give 2-3 seconds of silence here.** Let the viewer read the narratives on screen before the voice continues.
6. **On "plain English narratives"** → slowly scroll through the generated commentary
7. **On "One click"** → stop scrolling, pause

### [TIMING NOTE]:
- This is the jaw-drop feature. Build anticipation with "one of the most useful features."
- The 2-3 seconds of silence while narratives are visible is critical. The viewer needs time to read.
- If the narratives look particularly good, you can read one line aloud in a separate short clip to insert here (optional).
