## 1. Video Overview — What We Are Producing

### Format

A screen recording with voice-over narration (your voice). You will walk viewers through the P&L Automation Toolkit, showing real actions running in real time. The goal is to make every Finance & Accounting employee excited about what this toolkit can do for them.

### Tone

- Professional but approachable — like a trusted colleague showing you something genuinely useful
- Confident and enthusiastic — you built this, you are proud of it, and it genuinely makes life better
- Practical — focus on "here's what this does for YOU" not technical implementation details
- Pacing — not too fast, not too slow. Pause after each demo to let the result sink in

### Structure Options

**Option A: Three Short Videos (Recommended)**
- Part 1: "What This Is and How to Open It" (5–6 min)
- Part 2: "The Power Features — Live Demo" (7–8 min)
- Part 3: "Advanced Capabilities and Getting Started" (5–6 min)

**Option B: One Full Video (18–22 min)**
- Complete walkthrough in a single recording

> **Recommendation:** Option A is better for attention span and shareability. Leadership can watch Part 1 and get the picture. Staff can watch all three. But Option B is simpler to record.

## 2. Pre-Recording Checklist

Complete every item on this list before hitting Record. Do not skip any step.

### Workbook Preparation

- [ ] Open the P&L workbook file (`ExcelDemoFile_adv.xlsm`)
- [ ] Confirm macros are enabled (no yellow security bar visible)
- [ ] Press **Ctrl + Shift + M** — verify the Command Center opens. Close it.
- [ ] Run **Action 45 (Quick Health Check)** — all 5 tests must show PASS
- [ ] Run **Action 7 (Scan for Data Quality Issues)** — verify the Data Quality Report is created with a letter grade
- [ ] Delete any leftover generated sheets from previous demos (Variance Analysis, Sensitivity Analysis, Executive Dashboard, etc.) so the viewer sees them being created fresh
- [ ] Navigate to the **Report-->** sheet (press Ctrl + Shift + H)
- [ ] Make sure the workbook looks clean — no extra windows, no scroll bars in weird positions
- [ ] If you want a "clean" demo, run **Action 48 (Executive Mode Toggle)** to hide technical sheets, then toggle it off — this confirms the feature works before recording

### Computer Preparation

- [ ] Close ALL other applications except Excel and your screen recorder
- [ ] Close all other Excel files — only the P&L workbook should be open
- [ ] Turn off all notifications (Teams, Outlook, Slack, Windows notifications)
  - Windows: Settings > System > Notifications > Turn off
  - Teams: Set status to "Do Not Disturb"
  - Outlook: Close it
- [ ] Close any browser tabs (avoid accidental tab previews in recordings)
- [ ] Set screen resolution to **1920x1080** (Full HD) for the clearest recording
- [ ] Make sure your desktop wallpaper is professional (plain color or company branded — nothing distracting)
- [ ] Plug in your laptop (battery mode can throttle performance)
- [ ] Close File Explorer and any other windows

### Audio Preparation

- [ ] Use a headset with microphone or a dedicated USB microphone (NOT the laptop's built-in microphone — it picks up fan noise and typing)
- [ ] Test your microphone: Record 10 seconds of yourself talking and play it back. Is it clear? Is the volume good? Is there echo or background noise?
- [ ] Find a quiet room with a closed door
- [ ] Turn off fans, AC units, or other noise sources if possible
- [ ] Have a glass of water nearby (talking for 20 minutes dries your throat)

### Script Preparation

- [ ] Read through the entire script below at least once
- [ ] Practice the key transitions (moving between actions) so you don't fumble during recording
- [ ] Know which actions you will demo and in what order
- [ ] Have this document open on a second monitor (or printed out) so you can glance at the script while recording

## 3. Equipment and Software Needed

### Screen Recording Software (Pick One)

| Software | Cost | Notes |
|---|---|---|
| **OBS Studio** | Free | Most popular. Download from obsproject.com. Records screen + audio simultaneously. |
| **PowerPoint Recording** | Free (included with Office) | Insert > Screen Recording. Simple but limited editing. |
| **Windows Game Bar** | Free (built into Windows 10/11) | Press Win+G to open. Quick and easy but basic. |
| **Loom** | Free tier available | Records screen + optional webcam. Easy sharing. |
| **Camtasia** | Paid (~$300) | Best editing features. Worth it if you plan to make more videos. |
| **Snagit** | Paid (~$63) | Good for shorter recordings with basic editing. |

> **Recommendation:** If you want something free and easy, use **OBS Studio**. If you want the best editing experience, use **Camtasia**. If you just need something quick, use **PowerPoint's built-in Screen Recording**.

### Recording Settings

| Setting | Value |
|---|---|
| **Resolution** | 1920 x 1080 (Full HD) |
| **Frame Rate** | 30 fps (standard) |
| **Audio** | External microphone (USB headset or desk mic) |
| **File Format** | MP4 (most compatible) |
| **Webcam** | Optional — a small webcam overlay in the corner adds a personal touch but is not required |

## 4. Screen Layout and Settings

### Excel Settings for Recording

Before recording, adjust Excel for the clearest visual:

1. **Zoom level:** Set to **100%** (View > Zoom > 100%). This ensures text is readable on all screen sizes.
2. **Ribbon:** Keep the ribbon visible (do NOT minimize it) — viewers need to see you are in Excel
3. **Formula bar:** Keep visible
4. **Status bar:** Keep visible (it shows progress during actions)
5. **Sheet tabs:** Make sure they are visible at the bottom
6. **Window:** Maximize Excel to fill the entire screen

### Font and Color Visibility

The workbook already uses company brand styling (Navy headers, white text, alternating row colors). This photographs well on screen. No adjustments needed.

## 9. Speaker Notes — Word-for-Word Script

### Part 1 Script: Introduction and First Impressions

**[0:00 — Opening. Excel is open on the Report--> sheet.]**

> "Hi everyone, I'm Connor from the Finance team. Today I want to show you something I've been building that I think is going to change the way we do P&L reporting.
>
> What you're looking at is the P&L Automation Toolkit. It's a single Excel file — this file right here — that automates our entire monthly P&L close process. No software to install, no subscriptions, no IT tickets. Just open the file and go."

**[0:30 — Scroll through sheet tabs at the bottom.]**

> "Let me show you what's inside. At the bottom here, you can see the workbook has multiple sheets. We've got the main P&L report, monthly trends, product line summaries, functional P&L breakdowns by department, reconciliation checks, and the Assumptions sheet where all the key drivers live.
>
> (Click on P&L - Monthly Trend) Here's the Monthly Trend — you can see revenue and expenses by month, all laid out cleanly.
>
> (Click on Product Line Summary) Here's the Product Line Summary — iGO, Affirm, InsureSight, DocFast — broken out individually.
>
> (Click on Checks) And here's the Checks sheet — this is where reconciliation results show up. We'll come back to this."

**[1:30 — Navigate back to Report--> sheet.]**

> "But here's the thing that makes this different from just a well-organized spreadsheet. Everything in this workbook is powered by a Command Center — a single control panel that gives you one-click access to 62 automated actions."

**[2:30 — Press Ctrl+Shift+M to open the Command Center.]**

> "Let me show you. I press Control-Shift-M...
>
> (Command Center opens) ...and here it is. The Automation Command Center. Version 2.1.
>
> On the left, you have categories — Monthly Operations, Analysis, Data Quality, Reporting, and more. On the right, you have the actions. 62 of them. You can scroll through them, or — and this is what I use most — you can just search."

**[Demonstrate the search box.]**

> "(Type 'variance' in the search box) If I type 'variance', it instantly filters to just the variance-related actions. (Clear the search) If I type 'PDF', it shows just the export options. It's like a search engine for financial automation."

**[3:30 — Run Action 45: Quick Health Check.]**

> "Let me run the first action. I'm going to do a Quick Health Check — this is Action 45. I click it, then click Run & Close.
>
> (Action runs, message box appears) And right there — 5 tests, all PASS. The workbook is healthy. This took 5 seconds. Doing this manually — checking every sheet, every formula, every reference — would take the better part of an afternoon."

**[4:30 — Run Action 7: Scan for Data Quality Issues.]**

> "Now let me show you something even more useful. I'm going to run a full data quality scan. (Open Command Center, select Action 7, Run & Close)
>
> (Data Quality Report sheet appears) Look at this. The system just scanned every sheet in the workbook for six types of data problems — text-stored numbers, blanks, duplicates, formula errors, formatting issues, and outliers.
>
> And right here at the top... (point to the letter grade) ...it gives you a Letter Grade. This workbook gets an A. That means zero critical issues. You can look at this and immediately know — the data is clean, the numbers are trustworthy.
>
> Imagine being able to open your P&L file on the first day of close and knowing, in 10 seconds, whether the data is ready to work with. That's what this does."

**[5:30–6:00 — Scroll through the Data Quality Report.]**

> "And below the grade, you have the full detail — every issue found, which sheet it's on, which cell, and what the problem is. Nothing is hidden. Everything is transparent."

>
